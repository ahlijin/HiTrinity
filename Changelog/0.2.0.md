# Changelog / 更新日志

## 0.2.0 (2026-06-10)

### 新增

- **三机 ROS2 代码实现**
  - `jetson/` — GPU 语音+视觉+融合工作空间
  - `pi5/` — 基站工作空间（心跳/LLM/TTS）
  - `pi3/` — 执行层工作空间（电机/LiDAR/里程计）

- **跨机消息定义 (trinity_msgs)**
  - 新包: CMakeLists.txt, msg/*.msg, srv/*.srv (9 msg, 2 srv)
  - FusionResult, Heartbeat, SystemMode, VoiceCommand, DetectionResult, ObjectInfo
  - LlmRequest (srv), LlmResponse (msg), TtsAudio, MapSync (srv)

- **融合节点 (fusion_node)**
  - `fusion_node.py` — 多模态融合节点
  - `mode_manager.py` — 模式管理器（Pi5心跳监听 + full/degraded切换）

- **GPU调度器 (gpu_scheduler)**
  - `gpu_scheduler.py` — 动态帧率调度（full 30fps / degraded 10fps / whisper 10fps / idle 5fps）

- **启动文件 (trinity_bringup)**
  - `jetson_bringup.launch.py` — Jetson 标准启动
  - `jetson_degraded.launch.py` — 降级模式启动
  - `trinity_full.launch.py` — 完整模式启动

- **Pi5 基站节点**
  - `pi5_heartbeat` — 1Hz 心跳发布
  - `pi5_llm_bridge` — Ollama LLM 推理桥接
  - `pi5_tts_bridge` — Piper TTS 音频桥接

- **Pi3 执行层节点**
  - `controller` — 电机控制（mecanum）+ 里程计发布
  - `peripherals` — LiDAR/IMU/遥控手柄
  - `ros_robot_controller` — 底层硬件驱动
  - `bringup` — 启动自检编排

### 改进

- **三机统一 ROS_DOMAIN_ID=42**，确保同一 DDS 域通信
- 启动脚本全部加 `chmod +x`，可直接执行
